## วิธีเรียกใช้ Skill

ใช้คำสั่ง `/` ตามด้วยชื่อ skill:

```
/strategist   → เรียกนักวางแผน
/brand        → เรียกนักออกแบบแบรนด์
/market       → เรียกนักการตลาด
```

### บนแต่ละเครื่องมือ (By Tool)

| เครื่องมือ | วิธีเรียก | ข้อควรรู้ |
|---|---|---|
| **Claude Code** | พิมพ์ `/ชื่อskill` แล้ว space ตามด้วยคำขอ | ต้องวางไฟล์ skill ใน `~/.claude/skills/` |
| **Cursor** | พิมพ์ `/ชื่อskill` ในช่อง AI Chat | ใช้ Custom Rules หรือ Project Rules |
| **ChatGPT** | ใช้ Custom GPT หรือพิมพ์ "ใช้ skill /ชื่อ" | ต้อง copy-paste เนื้อหา SKILL.md ไปใน prompt |
| **Gemini** | พิมพ์ "ใช้ /ชื่อskill" แล้วตามด้วยคำขอ | คล้าย ChatGPT ต้องวาง context ใน prompt |

## ลำดับการทำงาน

### 🗺️ แผนที่การเดินทาง (Business Journey Map)

```
เดือนที่ 1-2: วางรากฐาน
├── /strategist → รู้ว่าจะขายอะไร ใครซื้อ
└── /brand      → มีชื่อ มีภาพลักษณ์

เดือนที่ 2-3: สร้างช่องทาง
└── /build      → มีหน้าร้านออนไลน์

เดือนที่ 3-4: ดึงลูกค้า
├── /market     → ลูกค้ารู้จัก
└── /content    → มีเนื้อหาน่าสนใจ

เดือนที่ 4-6: ดูแลและขาย
├── /data       → รู้ว่าอะไรขายดี
├── /serve      → ลูกค้าติดใจ
└── /admin      → เอกสารเรียบร้อย

เดือนที่ 6+: เติบโต
├── /money      → รู้ตัวเลข กำไรชัด
└── /coach      → ปรับปรุงต่อเนื่อง
```

### ⚡ ใช้ย้อนกลับได้ไหม? (Can I Skip or Go Back?)

| คำถาม | คำตอบ |
|---|---|
| "ต้องทำตามลำดับเสมอไหม?" | **ไม่** — ใช้ skill ไหนก็ได้ตอนไหนก็ได้ แต่ลำดับนี้เป็น "แนวทางที่สุด" |
| "ธุรกิจเริ่มไปแล้ว ใช้ /strategist ได้ไหม?" | **ได้** — ใช้ตอนไหนก็ได้ แม้ธุรกิจจะเริ่มไปแล้ว |
| "ใช้หลาย skill พร้อมกันได้ไหม?" | **ได้** — เช่น "/market แล้วช่วยเขียนแคปชั่นด้วย" จะเรียก /content ช่วย |

## การสลับระหว่าง Skill

### 🔄 วิธีสลับ (How to Switch)

```
# กำลังคุยกับ /brand อยู่ แต่อยากให้ช่วยเรื่องการตลาด

ผู้ใช้: /market ช่วยวางแผนการตลาดร้านกาแฟของฉันหน่อย

AI: [สลับเป็นนักการตลาดทันที อ่าน context ร้านกาแฟจาก business_context แล้วตอบ]
```

### 📝 Context ที่ถูกส่งต่อ (What Gets Carried Over)

เมื่อสลับ skill สิ่งเหล่านี้ AI จะจำไว้เสมอ:
- ✅ ชื่อธุรกิจและประเภท (จาก business_context)
- ✅ สิ่งที่คุยกันมาในช่วงนี้ (session memory)
- ✅ เป้าหมายและสถานะปัจจุบัน (จาก project_context)

สิ่งที่ไม่ถูกส่งต่อ:
- ❌ รายละเอียดเฉพาะทางของ skill ก่อนหน้า (เช่น โค้ดสี hex จาก /brand ต้องบอกใหม่ถ้า /content จะใช้)

## การใช้หลาย Skill พร้อมกัน

### 🔗 Skill Chaining — ลูกโซ่ Skill

บางงานต้องใช้หลาย skill ร่วมกัน:

**ตัวอย่าง: เปิดตัวสินค้าใหม่**
```
/strategist → วิเคราะห์ตลาดสินค้าใหม่
     ↓
/brand      → ออกแบบแคมเปญชื่อและภาพ
     ↓
/content    → เขียนแคปชั่นเปิดตัว 5 ช่องทาง
     ↓
/market     → วางแผนงบโฆษณา
     ↓
/admin      → สร้างใบเสนอราคาโปรโมชั่น
```

**วิธีทำ**: บอก AI ว่า "ช่วยทำทั้งหมดนี้ให้หน่อย" หรือทีละขั้นก็ได้

## การเติม Context

### 🎯 ทำไมต้องเติม Context?

AI ที่ **รู้จักธุรกิจคุณ** จะตอบได้ตรงกว่า AI ที่ไม่รู้จัก 100 เท่า

### 📁 ไฟล์ Context ที่ควรมี

| โฟลเดอร์ | ไฟล์ | ทำอะไร |
|---|---|---|
| `core/` | `CLAUDE.md` | กติกาการทำงานของ AI |
| `business_context/` | `BUSINESS.md` | บริบทธุรกิจ — ขายอะไร ใครซื้อ แบรนด์เป็นยังไง |
| `project_context/` | `PROJECT.md` | บริบทโปรเจกต์ — เป้าหมายอะไร อยู่ตรงไหน ทำอะไรได้/ไม่ได้ |

### ⚡ วิธีใช้

1. **กรอก TEMPLATE.md** ในแต่ละโฟลเดอร์ (ตอนคำถาม)
2. **บันทึกเป็น BUSINESS.md / PROJECT.md**
3. **บอก AI ว่า** "อ่าน business_context ก่อนตอบ" หรือแนบไฟล์ไปกับคำขอ

> 💡 **เคล็ดลับ**: ยิ่งเติม context ละเอียด ยิ่งได้คำตอบที่ "เหมือนมีทีมงานรู้จักธุรกิจคุณจริง ๆ"

---

> 📚 **ไฟล์ที่เกี่ยวข้อง**:
> - `core/CLAUDE.md` — กติกาการทำงานของ AI
> - `business_context/TEMPLATE.md` — คำถามเติมบริบทธุรกิจ
> - `project_context/TEMPLATE.md` — คำถามเติมบริบทโปรเจกต์
